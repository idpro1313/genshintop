# Редакционный стандарт гайдов GenshinTop

Короткий контракт для материалов в [`info/guides/`](../info/guides/). Канон для публикации на сайте — тот же каталог (см. `ContentRepository`). Цель — связная проза, предсказуемая структура и единый тон справочника (не «лесенка» из одной фразы на строку).

## Тон и термины

- Спокойный нейтральный стиль, без капса и кликбейта в основном тексте.
- Бренд разработчика: **HoYoverse**; игра — **Genshin Impact** (допустимо «Геншин» в русском тексте осторожно, без смешения латиницы в одном слове).
- «Патч», «обновление», «баннер / молитва события», «примогемы», «смола» — по контексту; избегать устаревшего только «miHoYo» без упоминания HoYoverse в актуальных материалах.

## Структура статьи

1. **YAML frontmatter** — обязательные `title`, `category`, `sourceSlug`; по возможности `topic`, `status`, `gameVersion`, `audience`, `summary`, `date` / `updatedAt`, `reviewedAt`, `sources`, `relatedGuides`, `relatedCharacters`.
2. Один смысловой **H1 в теле допускается**, если шаблон сайта не дублирует заголовок; иначе начинать с вводного абзаца и `##`.
3. **Ввод** — 1–2 абзаца: о чём материал и кому полезен.
4. Сюжетные тексты — явный блок **«Спойлеры»** или предупреждение перед деталями.
5. Разделы — только **`##` / `###`**; списки — маркированные или нумерованные в Markdown; таблицы — по делу.
6. **Оглавление** — если статья длинная: список ссылок на заголовки (`[Раздел](#slug)`), без голых строк-плейсхолдеров.

## Антипаттерны миграции

- Не оставлять каждое слово или короткую фразу на отдельной строке (тичная вёрстка под узкий столбец).
- Не использовать пустые или якоря-заглушки `[текст](#)` без реального URL или заголовка.
- Не дублировать подряд одинаковые заголовки без содержания между ними.

## Мета и актуальность

- **`status`**: `active` — по-прежнему главный вход в тему; `dated` — привязка к версии всё ещё осмысленна; `historical` — архив, в тексте дать ссылку на актуальный материал.
- После существенной правки обновлять **`reviewedAt`** и при необходимости **`summary`** (до ~300 символов, без обрыва на полуслове).

## Внутренние ссылки

- Использовать относительные URL сайта: `/guides/...`, `/characters/...`, `/guides/banners` и т.д.
- При переименовании файла (slug) — см. [`docs/GUIDES_MERGE_SPLIT.md`](GUIDES_MERGE_SPLIT.md): редирект и обновление `relatedGuides`.
