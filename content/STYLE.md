# content/STYLE.md — редакционные стандарты

Этот документ заменяет удалённые `docs/GUIDE_EDITORIAL.md` и `docs/CHARACTER_EDITORIAL.md`. Применяется ко всем `*.md` под [`content/`](.).

## Тон и язык

- Русский язык, «вы»-обращение.
- Деловой, нейтральный тон. Без эмодзи, без восклицаний, без сленга.
- Заголовки H1 — только в frontmatter `title`; в теле статьи начинаем с H2.

## Структура файла

```markdown
---
<frontmatter по схеме из _templates/>
---

Один-два вступительных абзаца — без H1.

## Заголовок раздела H2

Параграфы 2–4 предложения. Списки — где они уместнее прозы.

## Связанные материалы

- [Название](/section/slug)

## Источники

- Имя источника, URL — дата обращения (для веб).
```

## Frontmatter — общие поля

| Поле | Обязательно | Тип | Назначение |
|------|:-----------:|-----|------------|
| `title` | да | string | H1 страницы |
| `section` | да | string | путь раздела (например `characters`, `guides/basics`, `news/banners`) |
| `slug` | да | string | стабильный URL-сегмент |
| `status` | да | `live`\|`draft`\|`stub` | стадия готовности |
| `summary` | да | string | 1–2 предложения для индексов и SEO |
| `gameVersion` | да | string | например `"6.x"` |
| `updatedAt` | да | YYYY-MM-DD | дата редакции |
| `sources.info[]` | нет | string[] | пути файлов из `info/` |
| `sources.archive[]` | нет | string[] | пути файлов из `archive/` |
| `sources.web[]` | нет | object[] | `{url, title, accessed}` |
| `related[]` | нет | string[] | slug или относительные пути в `content/` |

Доп. поля по типу — в шаблонах [`_templates/`](_templates/).

## Атрибуция

- Все веб-факты проходят через `sources.web[]` (URL и дата обращения) и блок «Источники» в конце статьи.
- Описания талантов, пассивов, эффектов сетов — **переформулировать**. Дословные цитаты из клиента и wiki запрещены.
- Цифры и формулы из веба указываются с осторожностью: «по данным фан-вики на дату X». При расхождениях между источниками — приводим оба или маркируем `status: draft`.

## Перекрёстные ссылки

- Внутренние ссылки — относительные внутри `content/`: `[Чжун Ли](/characters/zhongli)`.
- Запрещено ссылаться на slug, которого нет в репо. Перед ссылкой проверяйте наличие файла.
- В индексных страницах допустим раздел «Что планируется» с явным списком будущих slug — но без рабочих ссылок на них.

## Slug

- Только латиница `[a-z0-9-]`.
- Кириллица транслитерируется по упрощённой системе (`zhongli`, `kazuha`, `khu-tao`).
- Один slug — один URL за всю историю; смена slug = смена URL = редирект.

## Запреты

- Копипаста описаний из клиента, wiki или гайдов.
- Эмодзи в заголовках и теле (отдельный декор не нужен — таблицы и списки достаточно).
- Пустые разделы H2 без содержимого (если не успели — оставьте файл `status: stub` без таких разделов).
- Спойлеры основного сюжета без пометки в первом абзаце.
